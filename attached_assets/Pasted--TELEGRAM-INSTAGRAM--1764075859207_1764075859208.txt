ДОПОЛНИТЕЛЬНОЕ ТРЕБОВАНИЕ: КОНТАКТЫ ТОЛЬКО ЧЕРЕЗ TELEGRAM / INSTAGRAM

Очень важно: пользователь НЕ должен вручную вводить номер телефона в обычное текстовое поле.  
Контактные данные берём из Telegram-профиля (phone / username) или через отдельную верификацию.

На шаге «Контакты» (часть шага 3 "Основная информация") надо реализовать следующее:

1) Источник данных — Telegram WebApp initData
   - Попробовать получить:
     - номер телефона пользователя (если доступен),
     - username (например, @vasya).
   - Если есть номер телефона — показываем блок:

     Заголовок:
       "Как с вами свяжутся покупатели?"
     Вариант 1 (по умолчанию, если есть телефон):
       ▢ Использовать номер из Telegram: +375 XX XXX-XX-XX
          [Кнопка: "Использовать этот номер"]

   - Если телефон недоступен, но есть username:
       ▢ Связаться через Telegram: @username
          [Кнопка: "Использовать этот контакт"]

2) Если НЕТ ни номера, ни username в профиле:
   - Показываем простое, не давящее сообщение:
     - "В вашем Telegram-профиле не указан номер телефона и имя пользователя."
     - "Чтобы покупатели могли связаться с вами, рекомендуем добавить номер или username в настройках Telegram."
   - Краткая инструкция (общими словами, без жёсткой привязки к интерфейсу):
     - "Откройте настройки Telegram → профиль → добавьте номер телефона или имя пользователя."
   - При этом пользователь всё равно может продолжить публиковать объявление, но:
     - в карточке объявления будет отображаться только то, что у нас есть (например, Instagram, если он указан),
     - если ничего нет — можно показывать пометку вроде "Связь через чат в приложении" (если такая логика есть в проекте) или оставить без явного контакта.

3) Дополнительно — поле для Instagram
   - Добавить поле:
     - "Instagram (по желанию)"
     - placeholder: "@ваш_аккаунт"
   - Это поле МОЖНО вводить вручную.
   - Оно не обязательное, но если заполнено — передавать на backend и хранить в объявлении.

4) Стратегия приоритета контактов:
   - В первую очередь предлагаем:
     - номер телефона из Telegram (если есть),
   - если телефона нет, но есть username:
     - предлагаем username как основной контакт,
   - если нет ни того, ни другого:
     - не заставляем вводить телефон вручную,
     - мягко объясняем, как добавить номер/username в Telegram,
     - при этом даём возможность указать Instagram (как дополнительный канал связи).

5) В данных для backend (POST /api/ads):
   - НЕ отправлять "сырое" поле телефона из формы.
   - Отправлять структурно:
     - `contactType`: "telegram_phone" | "telegram_username" | "instagram" | "none"
     - `contactPhone`: номер, если взяли из Telegram-профиля,
     - `contactUsername`: @username, если используется он,
     - `contactInstagram`: Instagram-аккаунт, если введён.
   - Если выбран телефон из Telegram — это считается верифицированным источником.
   - Если выбран username — это также считается безопасным контактом (без SMS).
   - Никакие "левые" телефоны, введённые руками, в эту версию НЕ добавляем.

6) UI-нагрузка:
   - Всё должно выглядеть очень просто:
     - Крупный заголовок: "Как с вами свяжутся покупатели?"
     - Пара крупных блоков-выборов (телефон/username/Instagram),
     - Никаких сложных форм и мелких чекбоксов.
