3. UI для продавца / вкладки PRO: как использовать Trend + CategoryEvolution

Сделай план и (по возможности) черновой код для вывода этой аналитики в кабинете продавца (особенно во вкладке PRO, если она уже есть).

3.1. Для кого и где показывать

Основное место показа:

Кабинет продавца / магазина.

Если есть вкладка PRO — вынести туда как главный контент:

блок «Тренды спроса»,

блок «Динамика категорий».

Доступ:

как минимум для ролей seller, farmer, shop_owner.

если уже есть логика PRO-тарифов, можно часть виджетов делать доступной только PRO-пользователям (но это вторым этапом).

3.2. Виджеты из TrendAnalyticsService

Сделай компоненты на фронте (или спланируй их), например:

"Тренды спроса рядом"

Список 5–10 категорий/запросов, где спрос растёт.

Для каждой строки:

название категории/запроса,

стрелка ▲/▼ и процент изменения,

подсказка: «У вас нет объявлений в этой категории» / «У вас уже X объявлений».

"Лучшее время публикации" (если в данных есть time-series):

Heatmap/гистограмма: по дням недели и часам, когда лучше всего откликаются на объявления.

Минимально — простая табличка или текст: «Чаще всего откликаются в: вт, чт, 18:00–22:00».

3.3. Виджеты из CategoryEvolutionService

"Как меняется спрос по категориям":

График/диаграмма топ-категорий магазина:

ось X — время (недели/месяцы),

ось Y — просмотры / отклики / сохранения в избранное.

Либо упрощённый вариант:

список категорий с трендами «+15% к прошлой неделе» / «–8%».

Подсказки продавцу:

Если категория растёт и у продавца нет объявлений в ней → показать рекомендацию:

«Спрос растёт в категории “XXX”. Добавьте товар — вы будете в числе первых».

Если категория падает → показать текстовое предупреждение, но без драматизма:

«Спрос на “YYY” немного снизился, попробуйте обновить фото или цену».

Оформи это в виде списка конкретных React-компонентов, которые нужно сделать/доработать (с указанием файлов, если можешь распознать структуру проекта).

4. Показываем покупателям "горячие запросы" (HotSearchService)

Задача: вывести список горячих поисковых запросов в интерфейсе покупателя, чтобы:

показать людям, что сейчас популярно,

мягко направлять трафик в релевантные разделы/поисковики,

а продавцам потом давать подсказки, чего не хватает.

4.1. Где показывать hot search

Главный экран поиска / строка поиска:

Под полем ввода показать блок:

заголовок: «Сейчас ищут» или «Популярный поиск рядом».

чипы/кнопки с запросами: клубника, картошка оптом, детский стол, кроссовки и т.п.

При нажатии на чип:

подставить запрос в строку + сразу выполнить поиск.

Внизу главной ленты (опционально):

небольшой блок «Популярно в вашем городе» с 3–5 запросами.

Фильтрация по гео:

если HotSearchService поддерживает фильтрацию по региону/городу/радиусу — использовать её,

иначе — хотя бы глобальные тренды.

4.2. Формат API и UI

Ожидаемый ответ от HotSearchService (если нужно — добавь такой формат):

{
  "items": [
    { "query": "клубника", "count": 124, "region": "Brest", "trend": "up" },
    { "query": "детская одежда", "count": 98, "region": "Brest", "trend": "stable" }
  ]
}


На фронте:

Компонент HotSearchChips:

принимает items,

рендерит чипы с текстом query,

можно добавлять маленькие маркеры тренда (стрелка вверх/вверх-вбок).
