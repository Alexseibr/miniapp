2. ВВЕСТИ ЛОКАЛЬНУЮ ВАЛИДАЦИЮ
======================================

Считаем обязательными:
- categoryId (или categorySlug),
- subcategoryId,
- price.

Описание — опционально.

Сделай локальный стейт ошибок:

```ts
const [errors, setErrors] = useState<{ price?: string; category?: string; subcategory?: string }>({});
Функция валидации:

ts
Копировать код
const validate = () => {
  const nextErrors: typeof errors = {};

  if (!values.categoryId) {
    nextErrors.category = 'Выберите категорию';
  }
  if (!values.subcategoryId) {
    nextErrors.subcategory = 'Выберите подкатегорию';
  }
  if (!values.price || values.price <= 0) {
    nextErrors.price = 'Укажите цену';
  }

  setErrors(nextErrors);
  return Object.keys(nextErrors).length === 0;
};
Где values — текущие значения формы (из props/context).

Под каждым полем, если есть ошибка — показать маленький текст красным.

handleContinue:

ts
Копировать код
const handleContinue = () => {
  const isValid = validate();
  if (!isValid) return;

  // Сохранить значения этого шага в общий стейт мастера
  // Например, через onUpdateDraft / onChangeDraft:
  onUpdateDraft({
    categoryId: values.categoryId,
    subcategoryId: values.subcategoryId,
    price: values.price,
    description: values.description ?? '',
  });

  // Перейти на следующий шаг мастера

};
Важно:

onNext() или аналог уже должен быть в мастере для других шагов (локация, фото) — переиспользуй ту же механику.
