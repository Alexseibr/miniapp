верхняя часть (заголовок “Мой магазин” + вкладки Товары / Подача / Статистика / Спрос / PRO / Ярмарки)

Экран кабинета продавца сейчас выглядит примерно так:

Верх:

кнопка “назад”

заголовок “Мой магазин”

вкладки: Товары / Подача / Статистика / Спрос / PRO / Ярмарки

Контент текущей вкладки (список товаров / форма подачи / статистика и т.д.).

Низ:

глобальная нижняя навигация (та же, что на остальных экранах: Главная / Лента / Избранное / Профиль и т.п.).

Нужно:

Обернуть экран кабинета продавца в layout с тремя зонами:

Header (фиксированный):

стрелка назад

заголовок “Мой магазин”

табы: Товары / Подача / Статистика / Спрос / PRO / Ярмарки

Content (скроллируемый):

только содержимое активной вкладки

BottomNav (фиксированный):

нижние табы приложения (то, что уже используется глобально)

Запретить “проскролл” всей страницы браузера/Telegram WebApp и обеспечить, чтобы скролл был только внутри центрального контейнера контента.

2. Реализация на уровне React-компонента

Предположим, что есть экран типа SellerCabinetScreen / MyShopScreen.
Нужно:

Сделать корневой layout примерно в таком стиле:

// псевдокод, адаптируй к реальным именам компонентов/стилей
export const SellerCabinetScreen = () => {
  const [activeTab, setActiveTab] = useState<'products' | 'create' | 'stats' | 'demand' | 'pro' | 'fairs'>('products');

  return (
    <div className="seller-cabinet-root">
      <header className="seller-cabinet-header">
        {/* стрелка назад + заголовок */}
        {/* вкладки (табы) */}
      </header>

      <main className="seller-cabinet-content-scroll">
        {/* здесь переключаем контент по activeTab */}
      </main>

      <footer className="app-bottom-nav">
        {/* уже существующий BottomTabs / нижняя навигация */}
      </footer>
    </div>
  );
};


В стилях (CSS / styled-components / Tailwind — адаптируй к проекту):

Корневой контейнер должен занимать 100% высоты доступного окна (внутри Telegram WebApp):

.seller-cabinet-root {
  display: flex;
  flex-direction: column;
  height: 100vh; /* или 100% от доступной высоты миниаппа */
  overflow: hidden; /* важно: запретить прокрутку всего экрана */
}


Header (верхний блок с заголовком и вкладками):

.seller-cabinet-header {
  flex: 0 0 auto;
  /* тень/фон по желанию */
}


Нижняя навигация:

.app-bottom-nav {
  flex: 0 0 auto;
}


Центральный блок (скролл-контент):

.seller-cabinet-content-scroll {
  flex: 1 1 auto;
  overflow-y: auto;
  -webkit-overflow-scrolling: touch; /* для гладкого скролла на мобилках */
}


Главное правило:
overflow-y: auto должно быть у внутреннего контент-контейнера, а не у body или корневого #root.

4. Исправить поведение с “закрытием” приложения при скролле

В Telegram WebApp иногда возникает ситуация, когда:

при свайпе вниз/вверх может дергаться/скроллиться страница браузера, а не внутренний контейнер →
Telegram начинает сворачивать окно.

Чтобы этого избежать:

На уровне глобальных стилей:

запретить scroll для body / html, когда открыт WebApp:

html, body {
  margin: 0;
  padding: 0;
  overflow: hidden;
}


Внутри мини-аппа все экраны (включая кабинет продавца) должны:

жить внутри контейнера с height: 100vh (или 100% от Telegram.WebApp.viewportHeight),

иметь скроллируемый только внутренний контент (как описано выше).

Если в проекте уже есть общий Layout для миниаппа — переиспользуй его, просто убедись, что:

кабинет продавца не задаёт свои высоты так, чтобы ломать этот layout,

вкладки и нижняя навигация не участвуют в прокрутке.

5. Проверки

После правок убедись, что:

На экране кабинета продавца:

при свайпе вверх/вниз двигается только список/форма внутри вкладки,

верхняя панель “Мой магазин + вкладки” всегда видна,

нижняя навигация всегда видна.

При длительном скролле:

мини-апп не сворачивается,

не появляется “отскок” страницы браузера.

Переключение между вкладками:

не релоадит весь экран,

не сбрасывает нижнюю навигацию.

Сделай всё вышеописанное:
Приведи SellerCabinet / MyShop к layout с фиксированным верхом и низом и скроллом только внутри контента, чтобы пользователь при пролистывании товаров/подач/статистики не закрывал мини-апп и не терял доступ к вкладкам.
