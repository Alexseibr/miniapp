Сейчас ситуация такая:

В профиле/кабинете магазина есть вкладка «PRO» / «Про»,

Она либо не открывается, либо показывает заглушку/пустой экран,

Из-за этого непонятно, что там должно быть и почему ничего не работает.

Хочу, чтобы ты сам нашёл в коде весь функционал, который был задуман, и выдал понятный отчёт: что планировалось, что сделано, что сломано, чего нет.

роуты/страницы вида:

/shop/pro,

/seller/pro,

/profile/pro,

/pro.

Собери список файлов, связанных с PRO-вкладкой, с путями, например:

и т.д.

В каждом найденном PRO-компоненте:

Посмотри пропсы, названия стейта, названия хуков,

Найди файлы с pro, монетизация, tariff, subscription, Avito Pro, фермерский PRO и проверь, не описан ли там модуль.

Используй только то, что реально находишь в коде/доках (если что-то додумываешь — явно помечай как гипотеза).

3. Объясни, почему вкладка PRO сейчас «не работает»

Нужно техническое объяснение, а не абстрактное «не доделали».

Проверь, как сейчас рендерится PRO-вкладка:

Откуда она берётся в таб-навигации (условия if (isProAvailable), if (role === 'seller'), featureFlags.proEnabled и т.д.),

Есть ли там реальный контент или только заглушка.

Проверь, какие API используются (или должны использоваться):

Есть ли что-то вроде GET /api/pro/dashboard, /api/analytics/seller/:id, /api/pro/features и т.п.,

Реально ли эти эндпоинты существуют в backend (поищи по проекту pro / analytics / tariff),

Вызываются ли они во фронтенде или код закомментирован.

Отметь технические причины, по которым вкладка не работает:

Например:

вкладка скрыта условием (proEnabled = false),

компонент подключён, но всегда возвращает null,

нет нужного API эндпоинта,

запрос падает с 404/500,

не обработано состояние loading / error,

старый код больше не совпадает с новой моделью данных.

Выведи это блоком:

"Почему вкладка PRO сейчас не работает"

Сделай конкретный список причин по пунктам.

4. Дай план задач: с чего начать оживление PRO-модуля

Нужен понятный roadmap в виде списка задач, которые можно отдавать разработчику.

Сформируй минимальный MVP вкладки PRO на основе того, что уже заложено в коде/доках:

какие блоки/виджеты сделать обязательно в первой версии,

какие можно отложить на потом.

Разбей задачи на категории: frontend, backend, общие.

Пример структуры (но заполни реальным содержимым):

Frontend

Подключить вкладку PRO в кабинете магазина:

файл: ...

включить в таб-навигацию для ролей: seller, farmer.

Реализовать компонент ProDashboard:

показать KPI: просмотры, клики, избранное, конверсия и т.п.

подключить графики (если есть готовые компоненты).

Обработать состояния:

loading / error / empty,

показать подсказки, если данных ещё нет.

Backend

Реализовать/починить эндпоинт GET /api/... для PRO-данных.

Добавить фича-флаг proEnabled в конфиг, если нужен.

В конце выведи блок:

"Пошаговый старт"

С 5–10 шагами, в каком порядке лучше браться за работу (например:

только потом подключать реальную аналитику и т.д.).

Итого:
Задача — не переписывать PRO-вкладку прямо сейчас, а сначала сделать техразбор:

где она в коде,

что по задумке,

почему сейчас не работает,

и дать понятный список задач для запуска минимальной рабочей версии.
